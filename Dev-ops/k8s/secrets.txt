secrets:
=======================================
A Secret is an object that contains a small amount of sensitive data such as a password, a token, or a key. 
Such information might otherwise be put in a Pod specification or in a container image. 
Using a Secret means that you don't need to include confidential data in your application code.
Secrets are similar to ConfigMaps but are specifically intended to hold confidential data.
Opaque is the default Secret type if you don't explicitly specify a type in a Secret manifest. 
When you create a Secret using kubectl, you must use the generic subcommand to indicate an Opaque Secret type.

Create secret: way#1
=========================
kubectl create secret generic my-secret --from-literal=server.servlet.context-path="/test"

kubectl get secret

kubectl describe secret my-secret

It will show like:
Data
====
server.servlet.context-path:  5 bytes

View/Edit:
===============
kubectl edit secret my-secret

output:

apiVersion: v1
data:
  server.servlet.context-path: L3Rlc3Q=
kind: Secret
metadata:
  creationTimestamp: "2026-03-27T00:04:29Z"
  name: my-secret
  namespace: default
  resourceVersion: "2275"
  uid: f20eb4fd-c9f2-41bc-8397-36d87f553e9b
type: Opaque

> This is the base 64 encrypted data

Decrypt this using Linux command:

L3Rlc3Q=  >>>>>>  echo L3Rlc3Q= | base64 --decode

It will decode this

Create a base 64 encrypted key:

echo /welcome | base64
L3dlbGNvbWUK

echo L3dlbGNvbWUK | base64 --decode
/welcome

Use this in a secret file:

vi sec-ex-1.yml
=========================

apiVersion: v1
kind: Secret
metadata:
  name: my-secret-2
type: Opaque
data:
  server.servlet.context-path: L3dlbGNvbWUK

Create a deployment:
================================
vi dep-ex-1.yml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: microservice-deployment
  labels:
    app-name: spring-boot-app
spec: 
  replicas: 2
  selector:
    matchLabels:
      app-name: spring-boot-app
  template:
    metadata:
      labels:
        app-name: spring-boot-app
    spec:
      containers:
        - name: sp-boot-cnt
          image: devsecopstrainer/spring-boot-app:v1
          env:
            - name: server.servlet.context-path
              valueFrom: 
                secretKeyRef:
                  name: my-secret
                  key: server.servlet.context-path
          ports:
            - containerPort: 8080

Create a service:

vi svc-ex-1.yml
=================================
apiVersion: v1
kind: Service
metadata:
    name: spring-boot-svc
spec:
    type: NodePort
    selector:
        app-name: spring-boot-app
    ports:
      - port: 8080
        targetPort: 8080
        nodePort: 30009
